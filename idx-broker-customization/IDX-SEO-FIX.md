# Fix: duplicate "IDX Property Services" results in Google

**Problem (seen in `site:gadurarealestate.com`):** the IDX subdomain
`homes.gadurarealestate.com` was getting its **search / map / results pages
indexed** — all titled the generic **"IDX Property Services"** — so the same
listing showed up 3× (e.g. `/idx/map`, `/map/mapsearch` ×2). Individual listing
titles also read oddly: *"Residential for sale in ANCRAMDALE, New York, 922447"*
(the trailing number is the MLS#).

**Why it can't be fixed from the website repo:** these pages are served by IDX
Broker on the subdomain, not from this GitHub repo. The subdomain's `robots.txt`
and page `<title>`/meta are generated by IDX Broker. Fix is applied in the IDX
Broker dashboard.

---

## Do this (5 minutes, in the IDX Broker dashboard)

### 1. Paste the SEO fix (primary fix)
- Open **Designs → (your active design) → Subheaders**.
- Scope: **Global**. Turn **WYSIWYG OFF** (so it pastes as raw HTML).
- Paste the entire contents of **`idx-seo-fix.html`** (in this folder).
- Save. This adds `noindex,follow` to every IDX page that is **not** an
  individual listing (map, mapsearch, results, search, featured, account…),
  and rewrites listing-page titles to the **street address** + a self-canonical.

> Google renders JS and honors a JS-set `robots`/`title`; the duplicate
> "IDX Property Services" results will drop over the next few crawl cycles.

### 2. Check the native SEO setting (belt-and-suspenders)
- In IDX Broker, look under **Account / SEO** (or **Designs → SEO**) for a
  **page-title template** and any **"index search results"** toggle.
- Set a real title template for detail pages (e.g. `[address] – [city], NY | Gadura Real Estate`)
  and **turn OFF indexing of search/results pages** if the option exists.

### 3. (Optional, for an instant drop) Search Console removal
- Add **`homes.gadurarealestate.com`** as its own property in Google Search
  Console (separate from the main domain).
- Use **Removals → Temporary removals** on URL prefixes
  `https://homes.gadurarealestate.com/idx/map` and `/map/mapsearch`.

---

## What stays indexed vs. dropped
| Page type | Example | Result |
|---|---|---|
| Individual listing | `/idx/details/listing/c056/123…` | **Indexed**, titled by address |
| Map / search / results | `/idx/map`, `/map/mapsearch`, `/idx/results` | **noindex** (dropped) |
| Account / login / saved | `/idx/login`, `/idx/saved…` | **noindex** (dropped) |

The main site's own listing pages (`gadurarealestate.com/homes/…`) are
unaffected and remain the primary SEO surface.
